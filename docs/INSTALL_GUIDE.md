## 2. 快速安装

### 2.1 使用安装包（推荐）

1. **下载安装包**
   
   从发布页面下载最新版本的安装包：
   ```
   IndustrialDefectDetection_Setup_v2.2.0.exe
   ```

2. **运行安装程序**
   
   双击下载的安装包文件，启动安装向导。

3. **选择安装类型**
   
   选择「快速安装（推荐）」选项，点击「下一步」。

4. **选择安装目录**
   
   建议使用默认安装路径，或点击「浏览」选择自定义路径：
   ```
   C:\Program Files\工业表面缺陷智能检测系统
   ```

5. **等待安装完成**
   
   安装程序将自动完成以下操作：
   - 安装必要的运行时环境
   - 复制项目文件
   - 创建快捷方式
   - 配置系统参数

6. **启动程序**
   
   安装完成后，可选择「立即启动」复选框启动程序。

### 2.2 快速启动

安装完成后，系统将自动创建以下启动方式：

| 启动方式 | 位置 | 说明 |
|----------|------|------|
| 桌面快捷方式 | 桌面 | 双击图标启动 |
| 开始菜单 | 开始菜单 → 程序 → 工业表面缺陷智能检测系统 | 包含多个启动选项 |
| 开机启动 | Windows启动文件夹 | 可选配置自动启动 |

## 4. 环境配置

### 4.1 环境变量

安装程序将自动配置以下环境变量：

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| JAVA_HOME | Java安装路径 | `C:\Program Files\Java\jdk-17` |
| NODE_HOME | Node.js安装路径 | `C:\Program Files\nodejs` |
| PYTHON_HOME | Python安装路径 | `C:\Python311` |

### 4.2 端口配置

系统使用以下默认端口：

| 端口 | 服务 | 说明 |
|------|------|------|
| 3000 | 前端服务 | Next.js开发/生产服务器 |
| 8080 | 后端服务 | Spring Boot API服务 |
| 8000 | AI服务 | FastAPI推理服务 |

如需修改端口，请编辑以下配置文件：

```
安装目录\config\.env
```

修改以下配置项：

```env
# 前端端口
NEXT_PUBLIC_API_URL=http://localhost:8080

# 后端端口
SERVER_PORT=8080

# AI服务端口
AI_SERVICE_PORT=8000
```

### 4.3 数据库配置

#### 4.3.1 使用内置SQLite（默认）

默认使用SQLite数据库，无需额外配置：

```env
DATABASE_URL="file:./data/platform.db"
```

#### 4.3.2 使用MySQL（可选）

如需使用MySQL数据库：

1. 安装MySQL 8.0+

2. 创建数据库：
   ```sql
   CREATE DATABASE industrial_defect_detection CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

3. 修改配置文件：
   ```env
   DATABASE_URL="mysql://username:password@localhost:3306/industrial_defect_detection"
   ```

4. 执行数据库初始化：
   ```bash
   npm run db:push
   ```

## 5. 服务启动

### 5.1 启动方式

#### 5.1.1 使用快捷方式

双击桌面图标或开始菜单中的启动项即可启动所有服务。

#### 5.1.2 使用启动脚本

在安装目录下有多个启动脚本：

| 脚本 | 功能 |
|------|------|
| `start-platform.bat` | 启动完整平台（前端+后端+AI） |
| `start-frontend.bat` | 仅启动前端服务 |
| `start-backend.bat` | 仅启动后端服务 |
| `start-ai-ml.bat` | 仅启动AI推理服务 |

#### 5.1.3 手动启动

**启动前端服务：**
```bash
cd 安装目录
npm run start
```

**启动后端服务：**
```bash
cd 安装目录\backend
java -jar target\backend-0.0.1-SNAPSHOT.jar
```

**启动AI服务：**
```bash
cd 安装目录\services\ai-ml
python main.py
```

### 5.2 启动参数

可在启动时指定参数：

```bash
# 指定端口启动
npm run start -- -p 8080

# 生产模式启动
npm run start -- -p 3000 --production

# 调试模式启动
npm run dev -- -p 3000
```

### 5.3 服务状态检查

启动后可通过以下地址访问：

| 服务 | 地址 | 说明 |
|------|------|------|
| 前端 | http://localhost:3000 | 主界面 |
| 后端API | http://localhost:8080/api | REST API |
| AI服务 | http://localhost:8000/docs | API文档 |
| 健康检查 | http://localhost:8080/actuator/health | 服务健康状态 |

## 7. 卸载程序

### 7.1 通过开始菜单卸载

1. 点击「开始」菜单
2. 找到「工业表面缺陷智能检测系统」程序组
3. 点击「卸载」选项
4. 确认卸载操作

### 7.2 通过控制面板卸载

1. 打开「设置」→「应用」→「已安装的应用」
2. 找到「工业表面缺陷智能检测系统」
3. 点击「卸载」按钮
4. 按照提示完成卸载

### 7.3 手动卸载

1. 停止所有运行的服务
2. 删除安装目录（默认：`C:\Program Files\工业表面缺陷智能检测系统`）
3. 删除数据目录（默认：`%APPDATA%\工业表面缺陷智能检测系统`）
4. 删除桌面快捷方式
5. 删除开始菜单文件夹
6. 清理注册表（可选）

### 7.4 卸载后清理

卸载后可选清理以下内容：

```powershell
# 删除数据目录
Remove-Item -Path "$env:APPDATA\工业表面缺陷智能检测系统" -Recurse -Force

# 删除日志目录
Remove-Item -Path "$env:LOCALAPPDATA\工业表面缺陷智能检测系统" -Recurse -Force

# 清理注册表（需管理员权限）
Remove-Item -Path "HKLM:\Software\工业表面缺陷智能检测系统" -Recurse -Force
```

## 8. 常见问题

### 8.1 安装问题

#### Q1: 安装程序提示"无法创建目录"
**解决方案**：确保目标磁盘有足够空间，且有管理员权限。关闭杀毒软件后重试。

#### Q2: 安装过程中断
**解决方案**：检查网络连接。尝试禁用防火墙或杀毒软件后重新安装。

#### Q3: 运行时环境下载失败
**解决方案**：手动下载运行时环境：
- [Node.js](https://nodejs.org/dist/v20.11.0/node-v20.11.0-x64.msi)
- [Java JDK](https://download.java.net/java/GA/jdk17.0.2/dfd4a8d0985749f896bed50d7138ee7f/8/GPL/openjdk-17.0.2_windows-x64_bin.zip)
- [Python](https://www.python.org/ftp/python/3.11.8/python-3.11.8-amd64.exe)

### 8.2 启动问题

#### Q4: 启动后浏览器显示空白
**解决方案**：
1. 检查端口占用：`netstat -ano | findstr 3000`
2. 等待服务启动完成（约30秒）
3. 清除浏览器缓存
4. 尝试使用其他浏览器

#### Q5: 后端服务启动失败
**解决方案**：
1. 检查Java环境：`java -version`
2. 检查端口占用：`netstat -ano | findstr 8080`
3. 查看日志：`安装目录\backend\logs\app.log`

#### Q6: AI服务无法启动
**解决方案**：
1. 检查Python环境：`python --version`
2. 安装依赖：`pip install -r requirements.txt`
3. 检查CUDA驱动（如使用GPU）

### 8.3 运行问题

#### Q7: 页面加载缓慢
**解决方案**：
1. 检查网络连接
2. 关闭不必要的浏览器标签页
3. 清除浏览器缓存
4. 考虑升级硬件配置

#### Q8: 数据不显示
**解决方案**：
1. 检查后端API是否运行：`curl http://localhost:8080/api/statistics`
2. 检查浏览器控制台错误
3. 查看服务日志

### 9.3 性能优化建议

1. **内存优化**：确保物理内存充足，避免同时运行大量程序
2. **网络优化**：使用有线网络连接，避免WiFi不稳定
3. **磁盘优化**：将数据和日志放在SSD上提升IO性能
4. **浏览器优化**：使用Chrome最新版本，定期清理缓存
